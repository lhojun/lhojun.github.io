    M1 맥 개발환경 설정 - (5) Jekyll 환경설정

    서론

    M1 Pro가 탑재된 2021 맥북프로 16’를 구매해서, 환경설정을 다시 할 필요성이 있었다. 이전 M1 맥북에어 환경설정을 할 땐 이리저리 충돌난걸 ‘야매’로 고쳐서 겨우겨우 돌아가게 한 것들이 많았기 때문에 이번 기회에 깔끔하게 설치도 해보고, 그 기록을 남기고자 한다.

    Jekyll 환경설정

    이 블로그는 Jekyll의 아주 유명한 테마인 Minimal Mistakes로 만들어졌다. 그리고 공부하는 식빵맘님의 글을 참고해서 카테고리와 태그를 만들었다. 카테고리와 태그는 저 분의 글에서 너무 잘 정리되어있고, 나는 Jekyll 로컬 개발 환경을 구축하는 글을 정리해두려고 한다.

    이 글은 아래 두 가지 과정을 진행했다는 가정하에 진행된다.

    Homebrew와 iTerm2가 안깔려있다면 꼭 아래 글들을 보고 깔고 오자!

    • M1 맥 개발환경 설정 - (1) 홈브루(Homebrew) 설치
    • M1 맥 개발환경 설정 - (2) iTerm2 설치 및 설정
      * iTerm2는 필수는 아니다! 맥의 기본 터미널에서 진행해도 똑같다!
    • brew를 통해서 rbenv와 ruby-build를 설치하자.

    rbenv는 ruby의 버전을 독립적으로 관리할 수 있게 도와주는 패키지이고,
    ruby-build는 rbenv의 플러그인으로, 서로 다른 버전의 ruby를 설치할 수 있게 도와주는 친구다.
    • 설치가 잘 되었는지 확인해보자
    rbenv versions

    • 현재 rbenv에서 사용할 수 있는 ruby버전들을 볼 수 있다.

    • 그 다음으로 현재 설치할 수 있는 ruby의 버전을 보자.
    rbenv install -l

    • 현재 설치 가능한 ruby버전들을 볼 수 있다.

    • 본인이 원하는 버전을 선택해서 설치하자.
    rbenv install 2.6.9

    너무 최신버전을 쓰면 안좋다는 나만의 편견에 따라 깔 수 있는 버전 중 가장 낮은 버전인
    2.6.9 버전으로 설치하도록 하겠다.
    • 설치가 완료되고 나면 다시 한번 사용할 수 있는 버전을 보자.
    rbenv versions

    • 방금 내가 설치한 버전이 있는것을 확인할 수 있다.

    • 이제 해당 버전을 전역으로 설정해주자.
    rbenv global 2.6.9

    global 대신 local을 써서 특정 디렉토리에만 버전을 구분 할 수 있다.
    나의 경우는 루비 버전을 굳이 따로 써야 할 일이 없기때문에 일단 전역으로 설정한다.
    • 그리고 ~/.zshrc에 rbenv의 내용을 반영해주어야 한다.
    code ~/.zshrc

    • 아래 내용을 복사해서 ~/.zshrc에 입력해주자.
    [[ -d ~/.rbenv  ]] && \
export PATH=${HOME}/.rbenv/bin:${PATH} && \
eval "$(rbenv init -)"

    저장과 터미널 재시작을 꼭 잊지말자!
    • 그 다음으로 bundler를 설치한다. 사실상 다 왔다.

    여기서 본인이 jekyll 프로젝트를 새로 시작하고 싶다면 같은 방법으로 'jekyll'도 설치해주자.
    나는 이미 있는 테마를 fork해서 사용할 것이기 때문에 따로 설치하지 않았다.
    • gem으로 무언가 설치했을때 rehash를 해주어야 한다고 한다.

    그런데, 사실 최근 버전들에선 안해도 된다고 하더라!
    이젠 gem으로 무언가가 추가, 삭제 되면 자동으로 rehash가 된다고 한다.
    이 TMI를 풀기 위해서 난 해주었다.
    • 블로그 프로젝트가 있는 폴더에서 bundler install을 해주자.
    bundler install

    나는 minimal mistakes 테마를 fork해서 폴더를 만들어놨다.
    그 폴더 안에서 bundler install을 통해 해당 블로그 테마에서 필요한 gem들을 한번 설치해주어야 한다.
    앞으로 포스팅을 하고, 디테일한 부분들을 변경할 땐 install을 따로 해주지 않아도 괜찮다.
    아마 대부분의 사람이 환경설정을 다시 하지 않는 한 다시 할 일이 없을 것이다.
    • install이 완료된 모습이다.

    • 이젠 로컬에서 실제로 구동해보자.
    bundle exec jekyll serve

    • 이렇게 뜨면 잘 된 것이다.

    Server address에 써있는 주소로 들어가보자. 블로그를 확인해 볼 수 있다.
    끝이다!

    참고로

    github에 올릴땐 serve한 상태로 올리면 안된다.
    bundle exec jekyll build해서 빌드를 해서 올리자. serve한 상태로 올리면 로컬 환경에서 구동된다고 설정되어 있어서
    링크가 로컬 주소를 갖게되는 현상이 발생한다.

    그리고, google analytics를 활성화 시키는 과정에서 알게 된 것인데,
    이 테마만 그런건지 모르겠으나, (아마 대부분 그렇겠지?)
    google analytics가 production일때만 활성화되어있게 설정되어 있기 때문에
    google analytics를 추가한 이후론
    JEKYLL_ENV=production bundle exec jekyll build를 통해 빌드를 해서 그 파일을 올려야한다.
    아마 빌드 옵션을 미리 설정할 수 있는 곳이 있을텐데.. 찾기 귀찮으니까 찾으면 다시 수정하는걸로
