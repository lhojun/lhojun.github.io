    M1 맥 개발환경 설정 - (3) React Native 개발 환경 설정

    서론

    M1 Pro가 탑재된 2021 맥북프로 16’를 구매해서, 환경설정을 다시 할 필요성이 있었다. 이전 M1 맥북에어 환경설정을 할 땐 이리저리 충돌난걸 ‘야매’로 고쳐서 겨우겨우 돌아가게 한 것들이 많았기 때문에 이번 기회에 깔끔하게 설치도 해보고, 그 기록을 남기고자 한다.

    React Native 개발 환경 설정

    동아리에서 React Native로 앱을 개발한지 어언 1년이 지났다. 그 사이에 맥북을 3번이나 바꾸었는데, 언제나 가장 골치 아픈 부분은 React Native 환경 설정이었다. 누가 꼭 방해라도 하는 것 마냥 제대로 되지 않고, 심지어 M1으로 와서 그 난이도가 30배쯤 어려워졌다. 하지만 이제 수많은 실패 끝에 React Native 환경 설정로 학사 학위를 딸 수준 정도는 되었다고 생각하기 때문에 글로 정리해두려 한다.

    이 글은 아래 두 가지 과정을 진행했다는 가정하에 진행된다.

    Homebrew와 iTerm2가 안깔려있다면 꼭 아래 글들을 보고 깔고 오자!

    • M1 맥 개발환경 설정 - (1) 홈브루(Homebrew) 설치
    • M1 맥 개발환경 설정 - (2) iTerm2 설치 및 설정
    • 그 어느 것보다 먼저 해야 할 일이 있다.

    바로 Xcode 설치(앱스토어에서)와, 시뮬레이터(에뮬레이터) 다운로드이다.
    이 두가지가 이 모든 과정 진행시간의 90%이기 때문에 먼저 진행한다.
    Xcode를 다운로드 후 실행하면 나오는 이 화면에서 커맨드(⌘ Command) 키와 쉼표(,)키를 눌러서 설정으로 가주자.
    • 에뮬레이터를 다운로드하자.

    열린 설정 창에서 Components탭으로 가면 에뮬레이터를 깔 수 있다.
    나 같은 경우는 이미 환경설정이 완료된 상태에서 다 지우고 다시 하는 것이기 때문에 이미 15.0 버전이 깔려있다.
    차마 시뮬레이터까지 다시 깔기는 너무 귀찮았다.
    아무튼 좌측의 다운로드 아이콘을 눌러 다운로드를 진행해두고 아래 과정을 진행하자!
    • nvm을 설치해주자.
    brew install nvm

    NVM; Node Version Manager
    노드의 버전을 유동적으로 사용할 수 있게 도와주는 아주 좋은 친구다.
    사실 일반적인 경우는 노드 버전을 굳이 바꿔가며 쓸 필요가 없겠지만
    나는 동아리의 개발 뿐만 아니라 가끔 Node.js로 프로젝트를 진행할 일이 있기 때문에 버전을 따로 관리하는게 편하더라.
    • 설치가 다 되면 여기서 해야 할 일이 있다.

    • ~/.zshrc 파일에 해당 내용을 추가해 주어야한다.
    export NVM_DIR="$HOME/.nvm"
[ -s "/opt/homebrew/opt/nvm/nvm.sh" ] && \. "/opt/homebrew/opt/nvm/nvm.sh"  # This loads nvm
[ -s "/opt/homebrew/opt/nvm/etc/bash_completion.d/nvm" ] && \. "/opt/homebrew/opt/nvm/etc/bash_completion.d/nvm"  # This loads nvm bash_completion

    일단 복사를 해두자. 본인 터미널에 출력된 것을 신뢰하자.
    • ~/.zshrc 파일에 추가하고 저장하자.

    들여쓰기를 고쳐서 넣어주자. 그냥 붙여넣으면 2,3번 라인이 한 수준 아래로 내려가더라.
    그리고 저렇게 주석으로 내가 무엇을 할 때 이 설정을 넣었는지 구분 해두면 미래에 편하다!
    • iTerm2(터미널)를 재시작하고, nvm을 입력해본다.
    nvm

    • nvm 커맨드를 알려주는 글이 출력된다면 성공적으로 설치된 것이다.

    • 그럼 설치 가능한 node의 버전을 보는 명령어를 입력해보자
    nvm ls-remote

    • 설치 가능한 버전들이 와르르 쏟아질 것이다.

    • 누가봐도 설치하고 싶은 가장 마지막 LTS버전을 설치하겠다.
    nvm install 16.14.0

    • 설치가 완료되면, node 16.14.0이 기본으로 사용되도록 바뀐다.

    • 아래 두 명령어를 통해 node와 npm의 버전이 잘 되어 있는지 확인하자.
    node -v
npm -v

    노드 설치가 완료되었으면 거의 끝난 일이라고 볼 수 있다.
    • watchman을 설치하자
    brew install watchman

    watchman은 파일 변경을 실시간으로 감지해주는 친구인데,
    리액트 네이티브에선 변경된 소스를 자동 빌드하고 시뮬레이터로 업로드하기 위한 용도이다.
    • 와르르 설치가 완료되고 나면, watchman 버전을 확인해서 설치가 잘 되었는지 보자.
    watchman --version

    • 버전이 날짜로 뜨는게 iTerm2인가 Homebrew인가의 버그였던가 그랬던 것 같은데 어쨋든 뜨기만 하면 된 것이다.

    • 다음으론 cocoapods를 설치해준다.
    sudo gem install cocoapods

    얘는 스위프트를 위한 의존성 관리 매니저이다.
    사실 이런거 몰라도 된다.
    비밀번호 입력을 해주자. (비밀번호 입력이 표시되지 않는 것에 당황하지 말자.)
    • 설치가 잘 되었는지 버전을 확인해보자.
    pod --version

    • 그 다음으로 ffi를 설치해주자.
    sudo gem install ffi

    얘는 아마 'Foreign Function Interface'의 약자일텐데
    리액트 네이티브의 어디에서 얘가 쓰이는지는 사실 잘 모른다.
    • 설치가 완료되고나면 드디어 대망의 React Native CLI를 설치해주자.
    npm install -g react-native-cli

    사실상 이 글의 가장 핵심
    • 설치가 완료되고나면 또 다시 버전 확인으로 설치가 잘 되었는지 확인하자.
    react-native --version

    사진이 없긴 한데, 버전이 뜨면 어쨋든 잘 된 것이다.
    사실 여기까지만 하면 끝이다. 프로젝트를 생성하고, 개발을 진행하면 된다.

    아주 골치 아픈 일이다.

    이거에 대해서 아주아주 많이 찾아보았는데.
    결론은 우리 동아리의 프로젝트에 포함된 패키지의 버전이 낮기 때문이다.
    찾아본 많은 글들에 의하면 우리 프로젝트의 react-native에 포함된
    node-sass의 버전이 노드 16버전을 지원하지 않고,
    내가 깐 노드의 버전이 16버전이라 발생하는 문제인데,


    문제는 M1, 즉 Apple Silicon을 노드는 16버전부터 공식적으로 지원했다는 것이다.
    이렇게 되면 해결법이 총 세가지로 나뉜다.


    첫 번째는 로제타로 어찌저찌 잘 해보는 것인데, 나는 이 방법은 매우 선택하기 싫었다.
    골치 아파가며 로제타로 억지로 돌리긴 싫었기 때문인데,
    로제타 써서 한 것 중에 한번에 제대로 된 것이 없는 과거도 한 몫 했다.


    두 번째로 할 수 있는 선택은 react-native의 버전을 올리는 것이었다.
    그래서 잠깐 우리 동아리의 프로젝트에 시도해봤더니 의존성때문에 라이브러리들이 와르르 에러를 내뱉더라.
    도저히 그 모습을 볼 수가 없어서 포기했다.
    사실 너무 귀찮았다!

    겨우 찾아 낸 마지막 해결법

    • 바로 ‘npm install’ 대신 이걸 쓰는 것이다.
    CXXFLAGS="--std=c++17" npm install
    그러면 놀랍게도 잘 된다.
    구형 node-sass의 소스에 표준 C++ 버전이 11로 되어 있어서 발생하는 버그인데,
    컴파일이 어쩌고 저쩌고.. 는 잘 모르겠고 17버전으로 강제해서 실행해주면 문제가 해결된다.
    • 실행까지 확인하기 위해 pod install을 하러가자.
    cd ios

    • pod install을 해주고
    pod install

    • 다시 프로젝트 폴더로 돌아와주자. 
      cd ..

    • 실행을 통해 잘 되는지 확인하자!
    react-native run-ios

    실행이 잘 되었다면 정말 환경설정 끝!
